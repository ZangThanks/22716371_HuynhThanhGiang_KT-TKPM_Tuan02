**KẾT LUẬN**
* State Pattern giúp quản lý trạng thái và hành vi tương ứng của đơn hàng.
* Strategy Pattern cho phép thay đổi thuật toán xử lý đơn hàng linh hoạt.
* Decorator Pattern giúp mở rộng hành vi xử lý đơn hàng mà không cần sửa đổi code gốc.\
  **=>** Kết hợp 3 pattern giúp hệ thống linh hoạt, dễ mở rộng và bảo trì.
